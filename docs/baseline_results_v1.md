# GOP 等价声学基线结果 v1

日期：2026-06-23

## 方法

由于当前机器没有预装 MFA、Kaldi 或深度语音模型，本版先使用 L2-ARCTIC 的人工
音素边界建立可复现的 CPU 基线：

1. 将 44.1 kHz 音频重采样为 16 kHz；
2. 为每个音素段抽取 24 维 log-Mel 特征和帧能量；
3. 使用 train 说话人的正确音素段训练 39 个对角高斯音素模型；
4. 计算目标音素与最强竞争音素的对数似然差，作为 GOP 等价分数；
5. 在 dev 说话人上校准音素组阈值；
6. 将 GOP 分数、目标/竞争似然、时长、音素、音素组和竞争音素输入带类别权重的
   Logistic Regression；
7. 在独立 test 说话人上一次性报告结果。

L2-ARCTIC Mandarin 只有 4 名说话人。本实验采用 BWC/LXC 训练、NCC 验证、
TXHC 测试，结果对说话人差异较敏感。

## 测试结果

| 指标 | 纯声学阈值 | 融合分类器 |
|---|---:|---:|
| Accuracy | 0.470 | 0.657 |
| Balanced Accuracy | 0.563 | 0.695 |
| Macro-F1 | 0.419 | 0.559 |
| AUC | 0.584 | 0.768 |
| 错误 Precision | 0.149 | 0.231 |
| 错误 Recall | 0.685 | 0.745 |
| 错误 F1 | 0.245 | 0.353 |

融合模型混淆矩阵，标签顺序为错误 0、正确 1：

```text
[[ 462,  158],
 [1539, 2783]]
```

## 与计划书验收线对照

- Balanced Accuracy：0.695，超过随机/多数类平衡基线 0.5 约 19.5 个百分点，
  距目标线 0.70 差 0.005；
- Macro-F1：0.559，达到最低线 0.55；
- AUC：0.768，达到最低线 0.70，尚未达到目标线 0.80；
- 错误 Recall：0.745，达到召回目标；
- 错误 Precision：0.231，未达到计划书要求的 0.40。

因此，本版完成了“可运行、可解释、可评估”的保底模型，但还不能判定基线模块完全
通过验收。主要问题是跨说话人误报过多。

## 产物

运行命令：

```powershell
python scripts/run_acoustic_baseline.py
```

生成目录：`artifacts/baseline_acoustic_v1/`

- `phone_gaussians.joblib`：39 个音素声学模型；
- `fusion_classifier.joblib`：融合分类器；
- `thresholds.json`：dev 集阈值；
- `metrics.json`：完整指标；
- `predictions.csv`：9,889 个 dev/test 音素预测。

## 下一轮优化

1. 为 SpeechOcean762 建立强制对齐，扩大普通话学习者训练说话人数；
2. 使用预训练自监督语音表征替代单高斯 log-Mel 模型；
3. 增加说话人/语句级归一化，降低录音和音色导致的误报；
4. 在 dev 集增加“错误 precision 不低于 0.40”的约束阈值，并在 test 上验证；
5. 分音素组分析误报，优先稳定样本量足够的 8–12 个核心音素组。
